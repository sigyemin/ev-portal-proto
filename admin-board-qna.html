DEV개발 참고 — 저장 구조·분류 코드·처리상태 (실서비스 비노출)
  • 저장 테이블 : APP_ARTC (BLBD_ID='app_qna') — 대민 질의응답과 동일 테이블. 목록/상세/답변 모두 이 게시판 기준.
  • 질문구분 코드 신설(복합키) : QNA_MAJOR(대분류 7) + QNA_MINOR(소분류 27, 부모 대분류 참조) → APP_ARTC에 대분류·소분류 2컬럼 신설. 대·소 복합키 필수(소분류 코드는 대분류 내에서만 유효). 기존 QNAGBN 단일 슬롯으로는 2단 수용 불가.
  • 기존 데이터 이관 : AS-IS SMLCLS10(전기차=1→ev / 저공해차=2→lev) 등 기존 분류를 신설 대분류로 매핑. 코드 미등록분(NULL 등)은 '기타'(etc)로 이관 후 운영자가 재분류. QNAGBN(답변구분)은 미사용.
  • 단일 원천 : js/qna-categories.js — 대민 질의응답 게시판(inquiry-qna·qna-write)·민원 처리 현황 조회(inquiry-status)와 동일 코드그룹 공유(변경 시 자동 일치).
  • 처리상태 : APP_ARTC.REPLY_STATE = 공통코드 CODE_SM(MI_ID=17) 4종 — 접수전(1)/처리중(2)/처리완료(3)/처리불가(4). AS-IS 2값(대기/완료) → 4종 확장. 접수전 요약행 = REPLY_STATE=1 집계.
  • 담당자(답변자) : 답변 저장 시 답변자가 수동 기입(로그인 사용자명 기본값). 대분류→담당팀 자동 매핑·자동 배정은 본 화면 범위 밖(별도 담당 배정 운영). 여기서는 매핑/자동배정 표기 없음.
