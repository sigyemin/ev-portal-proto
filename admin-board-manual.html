DEV개발 참고 — 신설 게시판·카테고리 코드그룹 (실서비스 비노출)
  • 신설 게시판 : 사용자매뉴얼 — APP_BLBD에 신규 행(BLBD_ID 신규 채번). 탭별 별도 게시판 확정(대민 자료실 4탭 구성과 무관 · 관리자에선 독립 항목). TNCM_MENU에도 신규 메뉴 등록 필요(시스템관리 > 게시판 관리, 서식다운로드 다음 순번).
  • 게시물 : APP_ARTC 공통 — ARTC_ID/BLBD_ID · 분류코드(카테고리) · 제목(매뉴얼명) · 내용(설명) · NM(게시물작성자 — 대민 '작성자' 컬럼 원천, 표시명) · INS_ID(등록자 계정 — 관리자 '작성자ID') · 공개여부 · 등록일/수정일 · 게시순서. 삭제는 DLT_YN.
    ★작성자는 관리자 직접입력이 아님 — 로그인 세션에서 자동 기입(NM = 로그인 사용자명, INS_ID = 로그인 계정ID). 등록 폼은 readonly. 수정 시에는 최초 등록자를 유지한다(수정자로 덮어쓰지 않음 — 필요 시 UPD_ID 별도 관리).
  • 첨부 : ATCH / ATCH_JFILE — ATCH_ID · ATCH_NM(파일명) · ATCH_SZ(용량 — 대민 '용량' 컬럼 원천) · ATCH_TYP · TGRT_ID = ARTC_ID. 다중 첨부.
  • ★카테고리 = 신설 코드그룹으로 DB 관리(하드코딩 금지) : 포털 이용 / 보조금 신청 / 공모 신청 / 충전기 설치·운영 / 회원·카드 5종. 코드ID·채번은 개발 재량(질의응답 신설 코드그룹과 동일 체계 권고). FAQ 분류 하드코딩(ARTC.SMLCLS10 + 라벨이 faq.jsp에 박힘) 안티패턴 회피 — 관리자에서 분류 추가·변경 가능해야 함.
  • 개정이력 : 별도 버전·이력 컬럼 없이 제목의 버전 표기(예: "…매뉴얼 v2.4")로 갈음.
  • 다운로드수 : 카운트만 집계, 개인 식별 로깅 없음. 어떤 매뉴얼이 실제 수요 있는지 파악용. 공개자료 다운로드 통계 연계는 선택.
  • 데이터 원천 공유 : 대민 누리집 자료실 > 사용자 매뉴얼 탭(info-library.html?tab=manual)과 동일 게시판 데이터 — 공개여부 Y 등록 시 즉시 대민 노출(단일 원천).
  • 범위 주의 : 업무포털(사업자·지자체)용 매뉴얼은 ev_ps MenualController 기존재 = 별개 트랙(이번 범위 아님).
