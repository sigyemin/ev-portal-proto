💳

보유한 회원카드가 없습니다

환경부 무공해차 통합 회원카드를 신청하면 전국 모든 충전소를 로밍 수수료 없이 이용할 수 있습니다.

회원카드 신청하기
차량번호 중복 안내 이미 다른 회원에게 발급된 차량번호로는 신규 발급이 제한됩니다(차량 1대당 1카드). 중복 시 발급이 불가하며, 자세한 사항은 누리집 콜센터(1661-0970)로 문의해 주세요. ※ 예외차량(중복확인 면제)은 기관 관리 영역으로 본 화면에서 등록·노출되지 않습니다.
🛠 카드상태 정의 (개발자용) — 단일코드 아님, 2축 우선순위 판정

표시 상태는 저장된 단일 상태코드가 아니라 2축 플래그를 resolveStatus()로 위에서부터 먼저 매칭해 결정합니다.

  • 액션 매핑: 발급조회=항상 · 분실신고=정상/발급중 · 정보수정=해지 아님 · 재발급=일시정지/미납/분실 · 해지=해지 아님.

🛠 개발 참고 — 영수증/이용내역 데이터 매핑 (2026-07-01 기획)

본 패널은 개발자용 참고입니다. 프로덕션 배포 전 제거 가능(HTML 주석 <details class="dev-note">).

이용내역(세션) 원천
EVCS_CHGE_TRADE_INFO(공단) ∪ R_TRADE_INFO(로밍/타CPO) — 기존 app_payment_sqlmap의 UNION ALL. 로밍: BID(이용사업자)≠TBID(충전기소유), 충전소명=R_STATION.CNAME·사업자명=R_BUSI.BNM.
세션 컬럼
STAT_ID/CHGER_ID, CHGE_STRT/END_DATE, CHGE_TOT_TIME, CHGE_TOT_POW(kWh), POW_FEE(충전요금), MON(결제금액), CARD_AUTH_NO(승인번호=영수증), POINT(이용포인트), CHGER_PAY_YN('N'=무료).
영수증 필드 매핑
충전기 CHGER_ID · 출력값=정격출력(kW, EVCS_CHARGER) · 타입=CHGE_TYPE_CD(01 DC차데모/02 AC완속/03 DC+AC3상)·로밍 R_CHARGER.CTYPE. 일시/양 CHGE_STRT_DATE·CHGE_END_DATE·CHGE_TOT_POW. 금액 POW_FEE·MON·POINT — 실단가=POW_FEE/CHGE_TOT_POW(할인 반영, 시간대 정가는 EVCS_CHARGE_PRICE.PRICE_00~23). 결제 PAY_KIND(대부분 4간편)·카드사 BILL_CARD_CD·끝4 BILL_CARD_NUM·승인번호 CARD_AUTH_NO(마스킹 없이 전체). 회원카드 CARD_NO(DB ARIA256 암호화·표시는 끝4)·CARD_TYPE(실물/모바일/회원번호/EV등록, 실코드 EVCS_CODE_DETAIL 'CARD_TYPE' 확인). ※충전사업자명=충전기 소유 사업자(R_BUSI.BNM/공단), 카드 발급소속(공단) 아님.
할인 · 부가세
거래에 할인 컬럼 없음 — 요금제 할인(EVCS_REMOTE_PRICE_PLAN.DISCOUNT_PRICE)은 POW_FEE에 이미 반영. BC그린카드 할인=카드사 대금청구 시(거래 밖). 영수증 할인·부가세 라인 없음(총액 표기).
결제방식·결과
PAY_KIND 1선불/2후불/3현장/4간편/5포인트 — 실질 대부분 4간편.
상태 정규화 (대민 6종)
충전중=CHRG_STAT=2(진행중, 미완료) · 완료=공단 POS PAY_RSLT=2 / 로밍 PAYRESULT=3 · 결제대기=공단 1(미결제) / 로밍 0·1·2·9(진행중이면 충전중 우선) · 결제실패=공단 3 / 로밍 4(사유=PERRMSG) · 취소=로밍 5(환불, 공단 없음) · 무료=FREE_YN='Y'/CHGER_PAY_YN='N'(PAYRESULT 아님). 원천: 공단 EV2_RAW_CHRGRPT.PAY_RSLT(1미결제/2완료/3실패) vs 로밍 R_BUSI_STTL_DETAIL.PAY_RESULT(0~9). 충전상태 CHRG_STAT(1대기/2충전중). ※요약 합계는 완료건만 — SUM(DECODE(CHGER_PAY_YN,'N',0,MON)) + 결과 필터.
명세(정산)
후불=EVCS_AFTER_PAYMENT(MON월·PAY_FEE입금액·PAY_DATE납부일·PAY_YN납부여부 Y/N=미납정지 근거·PAY_GUBUN·SET_SDATE/EDATE후불대상기간). 선불=EVCS_PRE_PAYMENT(DEPOSIT_AMOUNT/DATE/NAME/TOTAL), 잔여=입금−사용.
월별 명세서(PDF) 설계
신규 설계 — 기존 ev.or.kr에 충전요금 명세서 다운로드 없음(nportal 미추출로 참조 서식 없음). 명세서=결제 확정분 증빙(세션 상세 리스트 없음, 상세는 엑셀). 후불 근거 EVCS_AFTER_PAYMENT(MON·PAY_GUBUN 월회차·PAY_YN 자동결제 성공여부·PAY_DATE) + 로밍 R_BTRADE_DATA(YMON·MON·PAYEX 결제제외)·R_BTRADE_SCH(월 스케줄)·R_BTRADE_TOTAL. 합계=PAYEX/미결제 제외. “결제 완료”=배치 자동결제(승인) 성공(PAY_YN) — 사용자 카드대금 납부는 카드사 소관(우리 아님, ‘납부여부’로 표기하지 않음). 선불=EVCS_PRE_PAYMENT + 사용 세션 차감, 잔여=입금−사용.
CPO별 집계 · 제외 원칙
집계=충전기 소유 사업자(공단 EVCS / 로밍 R_TRADE_INFO.TBID→R_BUSI.BNM) 소계, 카드 발급소속(공단) 미표기. 제외: 결제실패·대기·취소·무료는 명세서 제외(상태로 이용내역/엑셀에서 확인). 부가세 분리 없음(총액)·포인트 정산 제외. 다운로드=인쇄뷰+window.print(사용자 ‘PDF로 저장’) / 정적 예시 PDF(assets/statement-sample-*).
포인트 (정산 제외)
명세 제외. 상세·잔액=EVCS_POINT(POINT_TOTAL/USED/REST)·EVCS_POINT_DTL(POINT_USE_CD 1적립·2사용·3사용취소·4적립취소) = 별도 포인트 페이지.
결제수단/빌키
EVCS_SMART_CARD_PAYMENT(BILL_KEY·BILL_CARD_CD·BILL_CARD_NUM끝4자리·PAY_YN). 빌키 발급=PG(이니시스/NICE/KICC). 카드신청(협회 대행)≠결제라인 자동생성 — 간편결제 등록 별도 스텝. 화면엔 카드사+끝4자리만(빌키 원문 노출 금지).
확정성
후불=정산 배치주기(매일 01:00~03:00 정산 수집, 월마감), 당월=잠정. 선불=입금(예치)/사용(차감).
탭 구분
탭=카드 과금모드(후불/선불, 후불대상설정 기준), 후불 default. 선불 발행이력 유지·사용 거의 없음(빈 상태 정상).
미확보 (실연동 확인)
충전사업자 사업자번호 · CARD_TYPE 실코드값 · 부가세 분리(소스에 없음→총액). 위 mock 표기는 확정값 아님(TODO).
