신청 상세

DEV개발 참고 — 공모 3종 상태값 (실서비스 비노출)
  • 사업수행기관 공모 : BRND_BSNS_STS — RS-01 신규 / RS-02 신청중 / RS-03 보완요청 / RS-04 재등록 / RS-05 검토완료. RS-01(신규)은 제출 전 상태라 대민 미노출(관리자 5종 · 대민 4종). 담당자 입력 = 보완내용 누적이력(SPLEMNT_CNTS, 날짜별 append) + 공단 메모(PBLCRP_MEMO) + 사업 메모(BSNS_MEMO).
  • 브랜드사업 공모 : APPR_STS — I 제출완료 / S 보완완료 / Y 승인 / N 반려 (T 임시저장은 목록 제외). 선정 별도 축 SLCTN_YN(Y 선정 / N 미선정, '-'=선정 전). 항목별 심사 = 신청 항목·첨부(27종)마다 승인/반려 CHK — 첨부 반려 = 대민 제출상태 '보완요청'. 담당자 메모 = APPR_MEMO.
  • BSS 공모 : APPR_STS 동일 4종 — 선정 축 없음(SLCTN_YN 부재, 승인이 최종). 항목별 CHK·APPR_MEMO 동일.
  • 메뉴 원천 : TNSM_BRND_MENU(메뉴+화면등록부 겸용). ⚠ 완속 직접신청(sc/directApply·sc/mgr — 매트릭스 A5) 메뉴는 이 테이블에 없음 — 하드코딩/별도 사이트 추정, 개발팀 확인 대상.
