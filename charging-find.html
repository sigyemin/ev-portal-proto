🛠 개발 참고 — 충전소 지도/마커 데이터 매핑

· ★마커 숫자 = 충전소 개수(2026-07-20 확정 — 충전기 대수 아님). 클러스터·멀티마커 숫자 = 그룹 내 충전소 수 · 단일 충전소 핀도 '1' 표시(모든 마커에 충전소 개수 노출) · 충전기 대수는 우측 슬라이드에서만 표시. 운영종료·폐쇄(DB 상태값) 충전소는 조회에서 제외 → 지도에 미표시.

· ★충전소 대표상태 = 저장값이 아니라 소속 충전기 상태에서 파생(§4-5 · AS-IS evMap.js:80~96). 산식 = 소속 충전기 중 우선순위 최상위 1개 채택 — 사용가능 > 충전중 > 점검중 > 통신미연결 > 상태미확인 > 운영중지. '전부 일치' 방식 아님(1기라도 사용가능이면 충전소는 사용가능). 사용가능 + 충전소 이용자제한 플래그 → 이용자제한 override. 프로토타입은 calcStationStatus(충전소ID) 공용 함수 1개로 일원화 — 마커·멀티 리스트·상세·다운로드가 전부 같은 함수 결과를 쓰므로 마커 ↔ 슬라이드 모순이 구조적으로 발생하지 않음. 더미도 대표상태를 하드코딩하지 않고 충전기 상태 구성(mix)만 정의 — 개발 시에도 대표상태 컬럼을 저장·이원 계산하지 말 것.

· 멀티마커(최종 LOD 전용) = 좌표 소수 4자리 반올림(≈10m) 근접 그룹에 충전소 2개 이상 → 클릭 시 슬라이드 1뎁스 충전소 선택 리스트(충전소명·CPO·대표상태 배지·충전기 N기) → 행 클릭 시 2뎁스 상세(← 목록 복귀). 충전소 1개 마커는 상세 직행. 표시 계층 병합만 — 충전소 코드 데이터 병합 금지(구역 분리·명의별 등록 혼재).

· 좌표 = LAT/LNG(예: 수소 H2_STATION.LAT/LNG, 전기 충전소 마스터의 위경도). 실시간 개별 충전기 상태는 로딩 과다로 지도 미노출 → 우측 슬라이드 전용.

· 클러스터 = 실서비스는 지도 라이브러리(네이버/카카오)의 as-is 자동 거리 클러스터(줌 축소 시 합침·확대 시 분할, 명시적 행정티어 없음). 클러스터 마커 = ★충전소 개수 합계(대수 아님).

· 연동: 필터(급속·사업자·회원카드 등) 조작 → 마커/클러스터 즉시 갱신 · 검색(주소·충전소명·지역) → 지도 이동+줌 · '내 주변' 리스트 ↔ 지도(같은 필터·데이터, 항목 클릭 → 마커 하이라이트+슬라이드). 본 프로토타입은 실지도 없어 mock 연동으로 시연.

· 즐겨찾기(자주 쓰는 충전소)는 로그인 회원별 → 본 페이지 '자주 쓰는 충전소' 뷰로 일원화(구 mypage-favorites 흡수). 비로그인 진입 시 로그인 유도.
